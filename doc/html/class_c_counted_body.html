Detailed Description

template<typename RT, RT NV, typename P1T, typename P2T, typename P3T, RT(*)(P1T, P2T, P3T) CF, void(*)(RT) DF>
class CCountedBody< RT, NV, P1T, P2T, P3T, CF, DF >

Implementação do RAII.

Usamos esta classe para implementar o RAII no gerenciamento do dispositivo Digital Persona (DP). A classe foi criada de forma mais genérica possível para viabilizar seu uso em outros casos. Futuramente ela deve sair daque e ganhar um contexto próprio (seus próprio .h).

Exemplo de uso:

Um dispositivo que e acessado atravez da funçao:

DPFPDD_DEV OpenDevice(char* szDeviceName, DPFPDD_PRIORITY pri);

Uma vez aberto, deve-se fechá-lo com a função:

CloseDevice(DPFPDD_DEV dev).

Criamos uma função para abertura do dispositivo compatível com a CCountedBody (3 parametros):

DPFPDD_DEV fnOpenDevice(char* szDeviceName, DPFPDD_PRIORITY pri, void* dummy)
{
return OpenDevice(szDeviceName, pri);
}

Criamos uma função para fechar o dispositivo

void fnCloseDevice(DPFPDD_DEV hDev)
{
CloseDevice(hDev);
}

Criamos então a classe que será, automaticamente, RAII:

class CDevice : public CCountedBody<DPFPDD_DEV, NULL, char*, DPFPDD_PRIORITY, void*, fnOpenDevice, fnCloseDevice>
{
private:
std::string _sContext;
public:
CDevice(char* szDeviceName, DPFPDD_PRIORITY pri = DPFPDD_PRIORITY_COOPERATIVE)
: CCountedBody(szDeviceName, pri, NULL)
{
}
protected:
virtual void swap(CDevice& s) throw()
{
CCountedBody::swap(s);
std::swap(_sContext, s._sContext);
}
};

Essa classe pode comportar-se como o próprio handle: Se o handle for usado para ser fechado, sem que seja chamado o destructor da classe, o destructor pode crashar ao tentar fechar um handle fechado. Resumo: não feche o handle externamente.
